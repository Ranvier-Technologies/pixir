# Operator Autonomy

Patch Operator is not full autopilot.

## The Skill May Do Without Asking

- Write or refresh evidence under `.docs/patches/<patch-id>/`.
- Narrow scope when evidence shows a donor change is `Reject` or `Defer`.
- Add rationale to the evidence ledger.
- Update `patch.md` when the change clarifies current scope without widening it.

## The Skill Must Ask First

- Widen patch scope.
- Change `preservePaths`.
- Delete evidence.
- Rewrite `AGENTS.md` or `CLAUDE.md`.
- Run external tools that may publish data, spend money, or depend on a subscription.
- Mark a patch `accepted` when required proof has unresolved gaps.

## Rule

Evidence creation and narrowing are autonomous. Scope widening, destructive actions, repo law changes, and external side effects require confirmation.
